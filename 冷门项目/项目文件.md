# 项目文件：Quantula 组队UI系统

## 项目概述

为 Minecraft 1.20.1 Forge 模组开发一个完整的组队管理UI系统，基于 ModernUI 框架实现。系统包含队伍创建、队伍列表浏览、队伍详情查看和管理等功能。

## 初始界面要求

系统启动时，首先显示 **TeamManagementFragment**（队伍管理主界面），该界面应根据玩家当前状态显示不同内容：

1. 如果玩家已在队伍中，显示一个空界面，界面中间有“已组建队伍”字样
2. 如果玩家不在队伍中，显示"创建队伍"和"浏览队伍"按钮

## 调用方式

系统通过以下方式集成到现有 Quantula 系统中：

### 1. 主入口点

```java
package xyz.ororigin.quantula.team_ui;

public class TeamUIEntry{
    //实现内容
}
```

### 2. Quantula系统启动界面方法：
```java
MuiModApi.openScreen(new TeamUIEntry())
```

## 界面设计要求

### CreateTeamFragment (创建队伍)

功能要求：

1. 队伍名称输入框（32字符限制，实时验证）
2. 队伍描述文本框（多行）
3. 颜色选择器（使用ColorPicker组件）
4. 创建按钮（输入验证通过后启用）
5. 错误提示区域
6. 取消/返回按钮


## 开发约束

### 1. 技术约束

- 所有界面必须继承自 `icyllis.modernui.fragment.Fragment`
- 使用 ModernUI 的 View/ViewGroup 体系
- 考虑 16:9 的宽高比限制
- 使用异步操作处理网络请求

### 2. UI/UX约束

- 保持一致的视觉风格
- 提供清晰的反馈（加载状态、成功/错误提示）
- 适配不同分辨率和GUI缩放

### 3. 错误处理要求

- 输入验证和错误提示

### 4. 项目结构约束

- 所有项目代码应该在xyz.ororigin.quantula.team_ui中

1. 不要修改现有的 `TeamManageUtils`实现，只需调用其API
2. 遵循ModernUI最佳实践
3. 保持代码结构清晰，添加必要的注释
4. 考虑模组的扩展性和可维护性
5. 注意ModernUI与Android并无关联，不能使用Android UI开发方法
